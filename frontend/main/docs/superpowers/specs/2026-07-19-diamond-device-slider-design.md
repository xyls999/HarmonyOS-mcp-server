# 设备页菱形滑轨设计

## 目标

将设备页的空调温度、风速、灯光亮度和窗帘位置从加减按钮改为可拖动滑轨。拖动期间只更新界面数值，松手或点击轨道完成后才向后端提交一次真实设备指令。

## 交互

- 空调温度范围 16–30℃，步进 1℃。
- 风速、亮度、窗帘位置范围 0–100%，步进 10%。
- 使用 ArkUI 原生 `Slider` 承担拖动、点击轨道和可访问性行为。
- `SliderChangeMode.Begin` 与 `Moving` 只更新本地预览值；`End` 与 `Click` 才调用设备 API。
- 离线设备或页面正在执行其他指令时禁用滑轨。
- 请求成功后使用设备快照校准；请求失败恢复拖动前的值并显示现有错误提示。

## 视觉

- 新建复用组件 `TacticalDiamondSlider`，轨道触控高度不少于 44vp。
- 滑块采用轨道内部的 45°菱形资源，深色主题为浅蓝，浅色主题为绿色；菱形不得伸出控件布局边界。
- 已选择轨道使用当前主题强调色，未选择轨道使用现有边框色；不引入新配色。
- 数值位于滑轨右侧并使用等宽视觉区域，拖动时不发生文字与布局跳动。
- 动效控制在 150–300ms，只表达拖动与提交状态，不添加循环装饰动画。

## 组件边界

`TacticalDiamondSlider` 只负责显示和交互，输入为 `value/min/max/step/unit/enabled/isDark`，输出为 `onPreview(value)` 与 `onCommit(value)`。设备 API 调用、乐观状态、失败回滚仍由 `DeviceCenterPage` 负责。

## 验收

- 设备页不再包含亮度、风速、窗帘和空调温度的加减按钮。
- 拖动过程不会重复发送网络请求；一次松手最多提交一次。
- 0、100、16、30 等边界值可达且不越界。
- 深浅主题下文字、轨道、菱形滑块清晰可见。
- Python 契约测试、ArkTS 编译、Release HAP 签名和 D6 真机拖动验证通过。
