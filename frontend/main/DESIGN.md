# Design System: Harmon Landscape Control

## 1. Visual Theme & Atmosphere
横屏 C 端语音主控屏。界面像顶级家庭中控屏: 语音是第一入口，触控只是补充。中心必须是大语音按钮、聆听状态、识别结果和执行反馈，其他场景、设备、统计都退为辅助信息。

## 2. Color Palette & Roles
- **Control Canvas** (#07110F) - 全局背景，避免纯黑。
- **Panel Surface** (#101C19) - 主卡片与模块表面。
- **Input Surface** (#16231F) - 输入框、胶囊与低层级控件。
- **Primary Signal** (#34C79A) - 唯一主强调色，用于选中态、发送、在线与关键状态。
- **Cold Text** (#F3FBF8) - 主文字。
- **Muted Text** (#B7C9C3) - 次级文字。
- **Dim Text** (#78908A) - 辅助信息与说明。
- **Structure Line** (#FFFFFF18) - 细边框。

## 3. Typography Rules
- 使用无衬线系统字体，保持设备端稳定渲染。
- 标题 30px 以内，避免营销页式大字。
- 数据数字使用更大字号和加粗权重，控制文案保持短句。
- 不使用 emoji，不使用装饰性大段说明。

## 4. Component Stylings
- **Navigation:** 左侧固定 rail，两个分页: 语音主控、统计信息。
- **Cards:** 8-22px 圆角，深色面板，弱边框，轻阴影。
- **Buttons:** 主操作使用 #34C79A，无霓虹外发光。
- **Inputs:** 深色底、清晰占位符、固定 44px 高度。
- **Charts:** 简洁柱状图，强调趋势，不展示过多日志。

## 5. Layout Principles
- 首页即实际主控台，不做落地页。
- 语音主控页以大麦克风为绝对中心，常用口令围绕它组织。
- 主控台仅集成辅助场景、快捷设备、环境状态和提醒摘要，不抢语音主视觉。
- 统计信息独立成页，只放能耗、环境、提醒摘要。
- 不出现用户中心、历史页、监控页等独立杂项入口。

## 6. Motion & Interaction
- 语音按钮必须有明显聆听态、按下反馈和状态文案。
- 仅保留选中态和卡片触摸反馈。
- 控制台以稳定为先，不使用大面积动画或漂浮装饰。
- 所有可点击控件至少 38-44px。

## 7. Anti-Patterns
- 不使用 AI 紫蓝霓虹渐变。
- 不使用三栏营销卡片叙事。
- 不使用“用户”“历史”“监控”“我的”等散页结构作为主信息架构。
- 不把 AI 做成普通聊天窗口。
- 不让统计卡、设备卡抢占语音主控的中心位置。
- 不使用无意义装饰文字、版本号角标、滚动提示。
- 不使用纯黑背景和高饱和多色混搭。
