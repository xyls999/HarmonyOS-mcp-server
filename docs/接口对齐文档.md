## 1. 核心规则

- **设备离线 = 关 + 没有数据**：不显示任何模拟/默认值
- **没有场景/模式**：回家/离家/观影/用餐等全部删除
- **检查到才确认**：只有真实从硬件读到数据才显示，否则显示"离线"
- **厨房1秒轮询**：按交接文档，alarm上升沿→BEEP ALARM
- **TTS只管输出(播报)**：语音输入由前端ASR→`/api/voice/input`
- **所有操作有日志+语音确认**

## 4. 厨房报警联动 (完全按交接文档)

### TCP轮询联动 (主通道, 1秒)
1. 中控每1秒查询一次厨房状态 (CMD4)
2. `alarm` 从 `0` 变为 `1` (上升沿) → 向客厅发送 `BEEP ALARM`
3. `alarm` 从 `1` 恢复为 `0` (下降沿) → 发送 `BEEP OFF`
4. **同一持续报警不重复触发蜂鸣器**

### UDP联动 (辅助通道)
- 厨房 UDP 使用 `255.255.255.255:8001` 局域网广播
- 不绑定某一台电脑，迁移后无需修改厨房固件
- 新电脑防火墙必须允许 UDP 8001 入站
- 交换机/路由器不能隔离无线客户端广播

### 厨房状态字段
| 字段 | 含义 |
|------|------|
| `smoke_level` | GP03 电平, 0=正常 |
| `smoke_alarm` | 1=烟雾报警(低电平报警) |
| `temp_alarm` | 1=热敏 ADC ≤ 1400mV |
| `alarm` | 1=烟雾或过热任一报警 |
| `thermal_mv` | GP07/ADC0 当前毫伏值 |
| `brightness` | 灯亮度 0-100 |

## 11. TTS职责划分

```
语音输出(播报): 后端TTS → channel.py tts_speak() → 百度TTS → oh_play播放
语音输入(识别): 前端ASR → POST /api/voice/input {text: "..."} → 后端处理
```

**TTS只管语音输出(播报)**，语音输入由前端ASR识别后发到 `/api/voice/input`。

## 12. 协议参考

### 客厅 (Hi3861, 文本协议)

| 命令 | 格式 | 回复示例 |
|------|------|----------|
| 温湿度 | `TEMP QUERY\n` | `OK,type=TEMP,temp=24,humi=51,alarm=NONE` |
| 事件 | `EVENT QUERY\n` | `OK,type=EVENT,seq=5,event=NFC_CARD` |
| 灯 | `LIGHT ON/OFF/QUERY\n` | `OK,type=LIGHT,state=ON` |
| 空调 | `AC ON/OFF/QUERY\n` | `OK,type=AC,state=ON` |
| 蜂鸣器 | `BEEP ALARM/OFF/QUERY\n` | `OK,type=BEEP,state=ALARM` |
| 门禁 | 二进制 CMD0(查询)/CMD1(设置) | `state=open/closed` |

### 厨房/卫生间/卧室 (H3863, 二进制协议)

```
包格式: AA 55 [CRC32 4B LE] [CMD 1B] [V1 1B] [V2 1B] [padding 21B] 55 AA
总长: 32字节
```

| CMD | 区域 | 功能 | V1 | V2 |
|-----|------|------|----|----|
| 4 | 厨房 | 状态查询 | - | - |
| 5 | 厨房 | 灯亮度 | 0-100 | - |
| 6 | 卫生间 | 状态查询 | - | - |
| 7 | 卫生间 | 灯亮度 | 0-100 | - |
| 8 | 卫生间 | 电机 | 方向(0/1/2) | 速度(0-100) |
| 9 | 卧室 | 状态查询 | - | - |
| 10 | 卧室 | 灯亮度 | 0-100 | - |
| 11 | 卧室 | 窗帘位置 | 0-100 | - |
| 12 | 卧室 | 窗帘动作 | 0=stop/1=home | - |
