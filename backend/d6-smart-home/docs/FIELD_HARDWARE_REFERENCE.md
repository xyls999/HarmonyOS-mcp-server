# A9 现场硬件权威参考（脱敏）

来源：`A9_现场启动脚本包_20260717`。本文件只保留后端和 AI 需要的硬件事实，不包含密码、共享密钥、Token 或私钥。

## 权威文件

- `connect/central_controller.py`：现场中央控制器和协议实现。
- `connect/devices.json`：设备地址、协议、雷达分区和安全策略。
- `connect/ac_ir_codes.json`：空调红外码本。
- 原现场包 `PROTOCOL_REFERENCE.md`：协议说明。
- 原现场包 `SM3_HMAC_边缘端鉴权对接说明.md`：边缘鉴权说明。

后端旧副本不得覆盖以上现场版本。设备端不新增 pip 依赖；中央控制器本身使用 Python 标准库，语音串口桥独立依赖 pyserial。

## 区域与设备

- 客厅：Hi3861，文本命令，灯、空调、蜂鸣器、门禁、温湿度和事件查询。
- 厨房：H3863，32 字节 AA55 二进制协议，灯、烟雾、热敏状态；另有 UDP 状态广播。
- 卫生间：H3863，32 字节 AA55 二进制协议，灯、换气扇、Rd-03 V2 毫米波雷达。
- 卧室：H3863，32 字节 AA55 二进制协议，灯和外接窗帘电机。

卧室外接窗帘的稳定能力是位置设置和停止；回零与全关可能受现场限位/安装状态影响，前端和 AI 不得把 `home` 描述成无条件可靠动作。

## 协议与边缘鉴权

- H3863 报文固定 32 字节：AA55、长度、命令、20 字节内容区、CRC32。
- Hi3861 使用 ASCII 命令；启用鉴权时携带 nonce 和认证标签。
- 边缘命令鉴权为 HMAC-SM3-TAG32 加 nonce。TAG32 是 HMAC-SM3 前 4 字节的小端整数。
- 这不等于端到端 TLS、SM2 或 SM4 加密。后端远程管理层的 SM2/SM3/SM4 与边缘设备鉴权是不同层次。
- 命令执行由中央控制器统一限频；硬件桥不得提前再次消耗同一限频窗口。

## 毫米波人体存在

- 雷达来源：卫生间 H3863 的 UART1，传感器 Rd-03 V2。
- 有效距离：20–110 cm。
- 区域：厨房 20–35 cm、卫生间 40–55 cm、客厅 60–85 cm、卧室 90–110 cm。
- 35–40、55–60、85–90 cm 的间隙是有意保留的未匹配区间。
- 滤波窗口 7 个样本，至少 5 个稳定样本才确认区域。
- `radar_presence.enabled` 是人体存在功能总开关，部署后保持 `true`。
- `radar_light.enabled` 是自动开灯联动开关。两者独立；修改其中一个不得隐式修改另一个。
- 现场雷达校准只有显式 `--apply` 才能写配置；雷达检测本身必须保持启用。

## 门禁规则

- `query` 只查询门状态，不需要密码，也绝不能被转换成关门命令。
- `open` 和 `close` 每次调用都必须携带用户本次手工输入的密码。
- 禁止从环境变量、数据库、历史对话、上次请求、日志、定时任务、场景、语音帧或模型输出补取密码。
- 密码只允许在当前调用栈短暂存在，不进入日志、SQLite、JSON、Prompt、RAG、操作记录或 MCP 审计。
- 自动场景和 AI/语音命令没有本次人工密码时，门禁子动作必须拒绝。
- 测试只验证缺少密码被拒绝、查询安全和脱敏；禁止在验收中真实开关门。

## 语音帧

现场语音桥可识别二进制 `AA 55 <category> <command> FB` 和旧式 ASCII 帧。原映射包含门禁、回家和离家动作；升级后的中央控制器对 open/close 使用显式密码策略，因此这些无密码语音门禁子动作会失败，不能读取预存环境变量自动通过。非门禁动作仍可按各自规则执行。

## 运维边界

- 不做整机重启测试。
- 不做真实门禁动作测试。
- 可做灯光的可恢复性控制测试：先读状态、执行一次、再恢复原状态。
- 部署前必须建立时间戳备份；启动后验证 `/health`、上下文 API、MCP initialize/list/read/call、毫米波开关和门禁拒绝路径。
